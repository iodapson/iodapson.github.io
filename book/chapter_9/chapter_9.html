9. The Rust concept that allows the existence of an enum that can encode the concept of a value being present or absent is represented by what enum?

Option<T>

10. What is the signature of the special Option enum?

Here:

  enum Option<T> {
    None,
    Some(T),
  }

11. Do you need to declare or import the Option enum in order to use it in your code?

No.

12. What does the Option enum allow you to do?

It lets you represent the concept of a value either being present or absent. It is a better replacement of Null.

17. Demonstrate code wherby you match to an Option pattern

Here:

  fn plus_one(x: Option<i32>) -> Option<i32> {
    match x {
      None => None,
      Some(i) => Some(i+1),
    }
  }
  let five = Some(5);
  let six = plus_one(five);
  let none = plus_one(None);

18. Specify how using the match construct can make using enums tedious

Rust knows when you don't cover every possible case and even knows which pattern you forgot (ie, the omitted arm). Matches in Rust are exhaustive; you must exhaust every last possibility in order for the code to be valid. Especially in the case of Option, when Rust prevents you from forgetting to explicitly handle the None case, it protects us from assuming that we have a value when you might have null, thus making the billion-dollar mistake of the 'null' constrct in other programming languages possible.

19. What can you do to avoid having to provision for every possible pattern of an enum, or even any other possible pattern of a non-enum value in a match construct?

By making use of a catch-all arm. Note that the catch-all arm must be the very last pattern inside your match construct.

20. Demonstrate code that specifies a catch-all arm inside a match construct

Here:

  let dice_roll = 9; // note that this is i32, the default when you do not specify the type for an integer value
  // now the match construct
  match dice_roll {
    3 => add_fancy_hat(),
    7 => remove_fancy_hat(),
    other => move_player(other), // this is the catch-all arm. You can name the placeholder anything asides 'other'. other's type must match the remaining patterns too.
  }
  fn add_fancy_hat() {}
  fn remove_fancy_hat() {}
  fn move_player(num_spaces: u8) {}
//The above code meets the exhaustiveness requirement of the match construct.

21. Demonstrate code that tells Rust explicitly that you aren't going to use any other value that does not match a pattern earlier specified in a match construct

  let dice_roll = 9;

    7 => remove_fancy_hat(),
    _ => reroll(),
  }

22. Demonstrate code that tells Rust explicitly that you aren't going to use any other value that does not match a pattern earlier specified in a match construct, and that it should not run any code in this case

23. What construct can be useful in situations where the 'match' expression is a bit wordy?

The if let ..syntax.

24. In what specific case would an if..let construct be appropriate, or considered for use in your code?

Whern you want a less verbose match alternative for code that should run when a value matches one pattern and ignores all other values. You want to sniff out a single pattern and run code for it.

25. Demonstrate code for matching against a single pattern value, while ignoring all others

Here:

  // Using the Option<T> enumeration
  let config_max = Some(3u8);
  match config_max {
    Some(max) => println!("The maximum is configured to be {}", max),
    _ => (), // this syntax (boilerplate code) makes it possible to skip writing code for other patterns
  }

26. Demonstrate code for matching against a single pattern value, while ignoring all others, using the if..let syntax

Here:

  // Using the Option<T> enumeration:
  let config_max = Some(3u8);
  if let Some(max) = config_max {
    println!("The maximum is configuredd to be {}", max);
  }

27. Demonstrate code that uses an else with a if..let syntax

Here:

  let mut count = 0;
  if let Coin::Quarter(state) = coin {
    println!("State quarter from {:?}!", state);
  } else {
    count += 1;
  }

28. Convert the code above in no-27 to a match construct

Here:

  let mut count = 0;
  match coin {
    Coin::Quarter(state) => println!("State quarter from {:?}", state),
    _ => count += 1,
  }
