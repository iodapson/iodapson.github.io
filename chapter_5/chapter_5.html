Programming a 'Guess the number Game'

1. By default, Rust has a few items defined in the standard libray that it brings into the scope of every program, what is the set of these items called in the Rust world?

The 'prelude'.

2. How do you create a new instance of an empty string in Rust?

With syntax:

String::new();

Example:

  let new_empty_string = String::new();

3. The 'new()' in syntax

String::new() ..is an example of what Rust concept?

Associated function. 'new()' is an associated function of type String.

4. What is an associated function?

It is a function that is implemented on a type. In the case of String::new() ..new() is an associated function of type Sting.

5. What syntax do you use to read user input from the command line?

Here:

std::io:stdin().read_line(bound_variable_ref).expect("Failure message, in case");

Ideally, you should import "std::io" into your program first, so you can simply skip the trailing "std::" to expose the "io::stdin()" part, as the new trailer.

6. Showcase syntax that imports the "std::io" library

Here: use std::io;

7. Demonstrate code that reads user input from the command line

Here:

use std::io;
fn main() {
  println!("I'll read your input as a string");
  let mut received_input = String::new();
  io::stdin
    .read_line(&mut received_input); // this line reads the user's input as a String, and then stores the input into a varible by referencing the variable. The referencer syntax itself '&' has been made mutable by means of syntax '&mut'
    .expect("Failed to read line"); // prints a message out in case of a read error in particular.
}

8. Does "io::stdin.read_line(bound_variable_ref)" return a value?

Yes it does.

9. What value does "io:stdin.read_line(bound_variable_ref)" return?

Here: io::Result

10. What is the nature of "io::Result"?

It is a submodule and an enumeration type, otherwise known as enum. "io::Result" is one the many submodule / enums type which Rust has.

11. What are Rust enums?

They are a concept or entity which have a fixed set of possibilities known as variants.

12. What are enums often used with?

Here: match

13. What is Rust's "match"?

It is a conditional that makes it convinient to execute different code, based on which variant an enum is when the conditional is evaluated.

14. What are the fixed set of possibilities or the variants that type "Result" has?

Two;

  • Ok
  • Err

15. How do method "expect("Failure message, in case") in syntax "std::io:stdin().read_line(bound_variable_ref).expect("Failure message, in case");" relate to the syntax itself?

Because "std:io::stdin.read_line" returns a Result, specifically type "io::Result" which has two variants "Ok" and "Err", method "expect" is actually a method of "io::Result" which would print out its string argument (the failure message) after crashing the program, if the "Result" variant evaluates to or is "Err". Whereas, if the variant is "Ok", then "expect" simply holds the value of the input too, without crashing the program.

16. What happens if you don't call "expect" when reading user input from the command line?

The program will compile, but Rust will issue a warning. Rust will warn that 'you haven't used the "Result" value returned from "read_line", indicating that the program hasn't handled possible error'.

17. What (third-party) crate can be used to generate random numbers?

rand

18. Imagine specifying crate "rand", version 0.8.3 as a dependency of your hypothetical project, as in:

rand = "0.8.3"

..what happens when you build the project for the very first time?

Cargo will download the "rand" crate and build your entire project together the crate.

19. Specify crate "rand", version 0.8.3, as in

rand = "0.8.3"

..as a dependency for your project, what will happen when you use the Cargo command "cargo update"?

Cargo will check to see if there is a newer version of "rand" up until version 0.9.0. If there is, it will update your crate, however, if there is any newer version than 0.90, it would not update it beyond version 0.9.0.

20. How do you update a third party crate to a version greater than 0.9.0?

By manually editing the version number inside file 'Cargo.toml', for example, you can change: rand = "0.8.3" ..to rand = "0.9.5" ..manually inside the file 'Cargo.toml', and then build your project so Cargo can download it and compile code components required from the crate.

21. Inside what file does Cargo store and keep track of third-party crate dependencies for yhour project?

Inside file 'Cargo.lock'.

23. Demonstrate code that generates a random number in the range of 1..101, and then stores such randomly generated number

Here:

use rand::Rng; // First import 'rand::Rng'
let rand_number = rand::thread_rng().gen_range(1..101);

24. What is the 'Rng' in the code line

use rand::Rng;

..called?

A trait.

25. What does the trait 'rand::Rng' do?

It defines methods that random number generators implement, and is a trait that must be in scope for us those methods.

26. What does code snippet "rand::thread_rng" do?

It is a generator for a particular random number that is local to the current thread of execution, and seeded by the operating system. After this snippet, you can then specify the range of random numbers you expect with syntax ".gen_rand(1..101)".

27. The method "gen_range()" is defined by what trait?

The "Rng" trait.

28. What alternate syntax can you use that is equivalent to "1..101"?

Here: 1..=100

30. What is the type of

std::cmp::Ordering

It is an enum.

31. List the variants that enum type "std::cmp::Ordering" has

  • Less
  • Greater
  • Equal

32. What does the method "cmp" of "std::cmp::Ordering" do?

It compares two values. It is a method that can be called on anything that can be compared.

33. How does method "cmp" work?

It take a reference to whatever you want to compare with, and then compare what you called it on, against it. It then returns a variant of enum "std::cmp:Ordering"

34. What is a "match" made up of?

arms

35. What is an enum "arm"?

An arm consist of a pattern to match against, and the code that should run if the value given to "match" fits that arm's pattern

36. Demonstrate code that uses the "match" syntax with method "cmp" called on something to compare against something else

Here:

match rand_number.cmp(&secet_number) {
  Ordering::Less => println!("Too small!"),
  Ordering::Greater => println!("tOO Big!"),
  Ordering::Equal => println!("You win!"),
}

37. Demonstrate code that cuts out white spaces including \n and \r from a string?

Here:

let the_string = the_string.trim();

38. Demonstrate code that parses a string into an u32 integer

Here:

let parse_target: u32 = parse_target.trim().parse().expect("sorry, it seems parse_target is a string that cannot be evaluated into a valid integer");

39. What does method "parse()" in code syntax

parse_target.trim().parse()

..return?

enum "Result". That is why method "expect()" was called on it in case variant "Err" was returned. In the case that variant "Ok" was returned, "expect()" would not return its error string argument. It would return the success value instead.

40. Demonstrate code that parse a string into an integer, and handles situations where there is either a successful parse/conversion or an unsuccessful one, using the "match" syntax

Here:

let target*number = match target_number.trim().parse() {
  Ok(num) => num,
  Err(*) => "There was an error",
};

41. What does the "" in syntax "Err()" in the code above, number 41, mean?

It is a catchall value that says match all "Err" values, which in this case, match all no matter what information they gave inside them.

42. Demonstrate code that creates a guess the number command line game

Here:

use rand::Rng;
use std:cmp::Ordering;
use std::io;

fn main() {
  println!("Guess the number!");
  let secret*number = rand::thread_rng().gen_range(1..101);

  loop {
    println!("Please input your guess.");
    let mut guess = String::new();
    io::stdin()
      .read_line(&mut guess)
      .expect("Failed to read line");
    let guess: u32 = match guess.trim().parse() {
      Ok(num) => num,
      Err(*) => continue,
    };
    println!("You guessed: {}", guess);

    match guess.cmp(&secret_number) {
      Ordering::Less => println!("Too small!"),
      Ordering::Greater => println!("Too big!"),
      Ordering::Equal => {
        println!("You win!");
        break;
      } // end of Ordering::Equal return-value block scope
    } // end of match block

  } // end of loop block

} // end of fn main
