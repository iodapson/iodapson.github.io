1. List three common collections available to you in Rust

  • vector; allows you to store a variable number of values next to each other
  • string; which is a collection of characters
  • hash map; which allows you to associate a value with a particular key. It's a particular implementation of the more general data structure called a map.

6. What instruments are available to you for reading an item from a vector?

  • indexing
  • get()

7. Demonstrate code that reads an item from a vector using both indexing and method "get()"

Here:

  let v = vec![1, 2, 3, 4, 5];
  let index_two = 2;
  // indexing approach
  let index_two_value: i32 = v[index_two.try_into().unwrap()]; // convert integer to usize, the valid indexing data type for vectors
  let index_three = 3;
  // get() approach
  let index_three_value: i32 = v.get(index_three.try_into().unwraps()); // convert integer to usize, the valid indexing data type for vectors.

NB: The get() syntax approch returns an Option enum, hence suitable for maching.

16. What syntax can you use to concatenate strings in Rust?

The + symbol/operator.

17. Demonstrate code that concatenates strings in Rust

Here:

  let s1 = String::from("Hello, ");
  let s2 = String::from("world!");
  let s3 = s1 + &s2; // note s1 has been moved here and can no longer be used. s3 becomes a new string

16. Describe one fundamental characteristic of Rust String concatenation syntax

It takes ownership of the String on its left-hand side, but not the one on its right-hand side. Hence, you must take ownership of the first string, while the second string must be borrowed (with &).

17. Rust String concatenation uses what syntax under the hood?

add()

18. Highlight the signature of add()

Here

  fn add(self, s: &str) -> String{} // where 'self' is a String

19. How is "add()" able to convert a &String (on the right-hand side of the concatenation) into an &str (its second argument)?

Deref coersion, which coerce the &String into &str, by turning the passed in s (&String) into &s[..] (it slices the &String into an &str).

20. String concatenation is a really good idea for joining many String data into a different and single String?

No. There is a better syntax to do just that. The better syntax is the "format" macro (as in, format!() ).

21. Demonstrate the better syntax to join many String data into a different and single String

Here:

  let s1 = String::from("tic");
  let s2 = String::from("tac");
  let s3 = String::from("toe");
  let s = format!("{}-{}-{}", s1, s2, s3);

22. How does the format!() macro specifically differ from the String concatenation syntax?

Unlike the concatenation syntax, the format!() macro does not take ownership of any of the Strings involved (i.e. passed in).

24. Does Rust allow you to target a single constituent of a String, by means of indexing e.g.

  let hello = String::from("hello");
  let answer = &hello[0];

No.

25. Why does Rust prevent you from being able to target/extract a String's constituent by means of indexing?

Rust sees/interprets and presents Strings to you the programmer from three angles namely; as an

  • a vector of bytes (u8 values, no more than 255)
  • array of scaler value (or Rust's char type)
  • grapheme clusters (what you might otherwise call letters, which don't include special characters like the comma ',' or question-mark '?' symbol).

Hence, You must provide more clarify on what you wish to target/extract from a String, beyond bytes returned by String indexing, which don't always return a valid scaler value at the very least.

26. What is the best way to target constituent values from a String?

Slicing. Slicing Strings is the best way to target/extract constituent values from a String.

27. Demonstrate code that is the best way to target/extract constituent values from a String, i.e. slicing the String

Here:

  let hello = String::from("Toughen");
  let s = &hello[0..5]; // s |> "Tough"

30. What is a Rust HashMap?

The type "HashMap<K, V>" stores a mapping of keys of type K to values of type V. It does this via a hashing function which determines how it places these keys and values into memory.

31. What is a hash map useful for?

For when you want to look up data not by using an index, as you can with vectors, but by using a key that can be of any type. For example, in a game, you could keep trak of each team's score in a hash map in which each key is a team's name and the values are each team's score. Given a team name, you can retrieve its score.

    scores.insert(String::from("Yellow"), 50);

33. Demonstrate code that creates a HashMap from two vectors (one supplying keys, while the other supplying corresponding values, respectively), powered by the "collect()" method

Here:

  use std::collections::HashMap;

  fn main() {
    let teams = vec![String::from("Blue"), String::from("Yellow")]; // this vector is chosen to supply keys
    let initial_scores = vec![10, 50]; // this vector is chosen to supply values
    let scores: HashMap<_, _> = teams.iter().zip(initial_scores.iter()).collect(); // the type annotation HashMap<_, _> is needed here because it is possible to collect into may different data structures and Rust doesn't know which you want unless you specify.
  }

  //In the code, the expression assigned to
  let scores: HashMap<_, _>
..Rust can infer the types that the hash map contains bases on the types of the data in the vectors.

40. What does method "or_insert()" return?

A mutable reference (&mut v) to the value for the key it is called on. That is why the code in no.39 above had to dereference the result 'count' from

  map.entry(word).or_insert(0);

41. What does the hashing function that HashMap use provide?

It is a cryptographically secure hashing function that provides resistance to Denial of Service (DoS) attacks, which is not the fastest hasing algorithm available, but offers better security performance.
